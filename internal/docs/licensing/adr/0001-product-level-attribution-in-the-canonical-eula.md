# Define Product-Level Attribution in the Canonical EULA

React on Rails Pro and its related components use one canonical EULA. The EULA
will define attribution as a single product-level obligation satisfied by the
part of the integrated product that emits the attribution; packages such as
`react-on-rails-rsc` that do not emit it have no independent obligation to add
it. We will update the canonical terms everywhere they are published instead of
creating an RSC-specific exception, avoiding divergent license rules across the
product suite. Runtime license metadata and diagnostic logs will report
attribution as required for every recognized plan: `paid`, `partner`, `startup`,
`oss`, `nonprofit`, and `education`. Free educational, demo, tutorial, workshop,
and personal or hobby use must also retain the generated attribution as a
condition of royalty-free use. For a valid license, the public attribution will
report `Licensed` without naming the licensed organization; organization
details remain in private server diagnostics. Expired, invalid, and missing
licenses retain distinct public status labels so compliance failures remain
diagnosable.

The approved substance for Section 6 is:

> Every HTML document containing output rendered by the integrated React on
> Rails Pro product must retain exactly one complete, unmodified, non-visual
> attribution comment generated by the product's attribution provider.
>
> The attribution provider emits that comment once on behalf of the integrated
> product. A related component, including `react-on-rails-rsc`, has no
> independent obligation to insert another comment when it does not control the
> HTML document or emit attribution.
>
> This requirement applies to paid Production Use, Complimentary OSS Production
> Use, and royalty-free uses permitted by Section 4.
>
> The generated comment includes its license-status suffix. A valid license
> displays `Licensed` without identifying the Organization. Expired, invalid,
> and missing licenses display `LICENSE EXPIRED`, `INVALID LICENSE`, and
> `UNLICENSED`, respectively.
>
> The licensee may not remove, modify, obfuscate, or disable the generated
> attribution.

`UNLICENSED` reports that no license credential was available to the renderer;
it is not by itself a legal conclusion that the use violates the EULA, because
Section 4 permits specified royalty-free uses without a license credential.
